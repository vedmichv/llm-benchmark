<task type="auto" tdd="true">
  <name>Task 2: Concurrent benchmarking module</name>
  <files>llm_benchmark/concurrent.py, tests/test_concurrent.py</files>
  <behavior>
    - run_concurrent_batch(model, prompt, n, timeout) fires N async requests via AsyncClient.chat() using asyncio.gather, measures wall_time via perf_counter, returns ConcurrentBatchResult
    - Failed requests get BenchmarkResult(success=False), other requests continue (gather does NOT use return_exceptions — each worker has its own try/except)
    - aggregate_throughput_ts = sum(all eval_count) / wall_time_s
    - avg_request_throughput_ts = mean of per-request response_ts for successful requests
    - benchmark_model_concurrent(model, prompts, num_workers, runs_per_prompt, ...) orchestrates: warmup once, then for each prompt x runs_per_prompt, call run_concurrent_batch
    - Per-request live output: "Request 1/4: 234 tokens @ 32.1 t/s" as each completes, then aggregate summary
    - auto_detect_concurrency(ram_gb, gpu_vram_gb) returns 2/4/8 based on resources
  </behavior>
  <action>
    Create llm_benchmark/concurrent.py with:

    1. `auto_detect_concurrency(ram_gb: float, gpu_vram_gb: float | None) -> int`: Returns 8 if gpu_vram_gb >= 16, 4 if ram_gb >= 32, else 2.

    2. `async _single_request(client, model, prompt, request_id, num_workers, console)` — inner async function. Calls client.chat(model=model, messages=[...]), converts response to BenchmarkResult via OllamaResponse. On exception, returns BenchmarkResult(success=False, error=str(exc)). Prints live per-request result as it completes: "Request {id+1}/{num_workers}: {tokens} tokens @ {rate:.1f} t/s" or "Request {id+1}/{num_workers}: FAILED - {error}".

    3. `async _run_batch(model, prompt, n, timeout)` — creates AsyncClient, starts perf_counter, launches N _single_request coroutines via asyncio.gather(*tasks), stops timer. Computes aggregate_throughput_ts = sum(eval_count for successful) / wall_time_s. Computes avg_request_throughput_ts = mean of per-request eval_count/eval_duration_s for successful. Returns ConcurrentBatchResult.

    4. `run_concurrent_batch(model, prompt, n, timeout)` — sync wrapper: calls asyncio.run(_run_batch(...)).

    5. `benchmark_model_concurrent(model_name, prompts, num_workers, runs_per_prompt, timeout, skip_warmup, verbose)` — orchestrates the full concurrent benchmark for one model. Calls warmup_model() once (from runner.py). Then for each prompt and run: calls run_concurrent_batch(). Collects all ConcurrentBatchResult objects. Prints aggregate summary line after each batch. Returns list of ConcurrentBatchResult. Uses get_console() for all output.

    Update tests/test_concurrent.py:
    - test_run_concurrent_batch: Mock AsyncClient.chat to return a canned response dict. Verify N results returned, wall_time > 0, aggregate_throughput_ts computed correctly.
    - test_failed_request_continues: Mock one chat call to raise, others succeed. Verify failed request has success=False, others have success=True.
    - test_aggregate_throughput: With known eval_count values and wall_time, verify aggregate_throughput_ts = sum(eval_count) / wall_time.
    - test_auto_detect_concurrency: Verify 2/4/8 thresholds.

    Important: Use ollama.AsyncClient (not raw httpx) per user decision. All terminal output through get_console(). The response from AsyncClient.chat() returns a ChatResponse object — use hasattr/model_dump pattern from runner.py to convert to dict for OllamaResponse validation.
