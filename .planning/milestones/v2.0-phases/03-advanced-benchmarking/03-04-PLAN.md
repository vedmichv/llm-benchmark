<task type="auto">
  <name>Task 1: Wire --concurrent, --sweep, and analyze into CLI</name>
  <files>llm_benchmark/cli.py</files>
  <action>
    Modify llm_benchmark/cli.py:

    1. In _build_parser(), add to run_parser:
       - `--concurrent`: type=int, nargs="?", const=-1, default=None. When flag present with no value, const=-1 signals auto-detect. When value provided, use that number. When flag absent, default=None means standard mode.
       - `--sweep`: action="store_true", default=False. Mutually exclusive with --concurrent (add mutually exclusive group for --concurrent and --sweep).

    2. Add analyze subparser to subparsers:
       - Positional arg: "file" (single JSON file path)
       - `--sort`: choices from VALID_SORT_KEYS, default="response_ts"
       - `--top`: type=int, default=None, help="Show top N models"
       - `--asc`: action="store_true", help="Sort ascending (slowest first)"
       - `--detail`: action="store_true", help="Show per-run breakdown"

    3. Create `_handle_analyze(args) -> int`: Lazy import analyze_results from llm_benchmark.analyze. Call analyze_results(filepath=args.file, sort_by=args.sort, top_n=args.top, ascending=args.asc, detail=args.detail). Return 0.

    4. Add "analyze" to _HANDLERS dict.

    5. Modify `_handle_run(args)`:
       - After preflight checks and before the model loop, check args.concurrent and args.sweep:

       **If args.sweep is True:**
       - Lazy import run_sweep_for_model from llm_benchmark.sweep
       - Lazy import export_sweep_json, export_sweep_csv, export_sweep_markdown from llm_benchmark.exporters
       - For each model: call run_sweep_for_model(model_name, timeout=args.timeout, skip_warmup=args.skip_warmup)
       - Collect all SweepModelResult objects
       - Export via sweep exporters (sweep_YYYYMMDD_HHMMSS.{json,csv,md})
       - Return 0 (skip normal benchmark flow)

       **If args.concurrent is not None:**
       - Lazy import benchmark_model_concurrent, auto_detect_concurrency from llm_benchmark.concurrent
       - Lazy import export_concurrent_json, export_concurrent_csv, export_concurrent_markdown from llm_benchmark.exporters
       - If args.concurrent == -1: auto-detect using system_info.ram_gb and system_info.gpu_vram_gb
       - Console print: "Concurrent mode: {N} workers"
       - For each model: call benchmark_model_concurrent(model_name, prompts, num_workers=N, runs_per_prompt=args.runs_per_prompt, timeout=args.timeout, skip_warmup=args.skip_warmup, verbose=args.verbose)
       - Collect all batch results, export via concurrent exporters
       - Return 0 (skip normal benchmark flow)

       **Else:** Existing standard benchmark flow (unchanged).

    Follow existing patterns: lazy imports inside handler functions, get_console() for output.
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Mode-aware exporters for concurrent and sweep results</name>
  <files>llm_benchmark/exporters.py</files>
  <action>
    Extend llm_benchmark/exporters.py with mode-aware export functions:

    1. Modify existing export_json: Add `mode: str = "standard"` to the data dict at top level. This ensures all standard exports include the mode field for forward compatibility. No other changes to existing function.

    2. `export_concurrent_json(batch_results: list[list[ConcurrentBatchResult]], system_info, output_dir="results") -> Path`:
       - Filename: benchmark_{timestamp}.json
       - JSON structure: {"generated": ..., "mode": "concurrent", "concurrent_workers": N, "system_info": {...}, "models": [{model, batches: [{prompt, num_workers, wall_time_s, aggregate_throughput_ts, avg_request_throughput_ts, results: [_result_to_dict(r) for r in batch.results]}]}]}
       - N is taken from first batch's num_workers

    3. `export_concurrent_csv(batch_results, system_info, output_dir="results") -> Path`:
       - Add Mode row in system info header: "Mode", "concurrent"
       - Columns: Model, Prompt, Workers, Wall Time (s), Aggregate (t/s), Avg Request (t/s), Per-Request details
       - One summary row per batch, optionally per-request detail rows

    4. `export_concurrent_markdown(batch_results, system_info, output_dir="results") -> Path`:
       - Header: "# LLM Concurrent Benchmark Results"
       - Mode note: "**Mode:** Concurrent ({N} workers)"
       - Summary table: Model, Prompt, Wall Time, Aggregate t/s, Avg Request t/s
       - Note at bottom: "Note: Ollama may queue requests. Aggregate throughput shows total work per wall-clock second."

    5. `export_sweep_json(sweep_results: list[SweepModelResult], system_info, output_dir="results") -> Path`:
       - Filename: sweep_{timestamp}.json (per user decision)
       - JSON: {"generated": ..., "mode": "sweep", "system_info": {...}, "sweeps": [{model, best_config: {num_ctx, num_gpu, response_ts, ...}, configs: [{num_ctx, num_gpu, response_ts, total_ts, ...}]}]}

    6. `export_sweep_csv(sweep_results, system_info, output_dir="results") -> Path`:
       - Filename: sweep_{timestamp}.csv
       - Columns: Model, num_ctx, num_gpu, Response (t/s), Total (t/s), Tokens, Time (s), Best?
       - "Best?" column: "Yes" for best config per model

    7. `export_sweep_markdown(sweep_results, system_info, output_dir="results") -> Path`:
       - Filename: sweep_{timestamp}.md
       - Per model: ranked config table, best row marked with "**Recommended**"
       - Bold the best config row

    Import ConcurrentBatchResult, SweepModelResult, SweepConfigResult from models at top of each function (lazy pattern not needed here since exporters is already a leaf module).
