<task type="auto" tdd="true">
  <name>Task 1: Analyze subcommand module</name>
  <files>llm_benchmark/analyze.py, tests/test_analyze.py</files>
  <behavior>
    - analyze_results("file.json", sort_by="response_ts") loads JSON, sorts models by averages[sort_by] descending, displays Rich table
    - analyze_results(..., ascending=True) sorts ascending
    - analyze_results(..., top_n=3) shows only top 3 models
    - analyze_results(..., detail=True) shows per-run breakdown rows under each model
    - Valid sort keys: response_ts, total_ts, prompt_eval_ts, load_time
    - load_time sorts by average load_duration_s across runs (computed from run data, not in averages)
    - Graceful error on file not found or invalid JSON
    - Table columns: Rank, Model, and then metric columns based on sort (always show response_ts, total_ts, prompt_eval_ts)
  </behavior>
  <action>
    Create llm_benchmark/analyze.py with:

    1. `VALID_SORT_KEYS = ["response_ts", "total_ts", "prompt_eval_ts", "load_time"]`

    2. `_compute_load_time_avg(model_data: dict) -> float`: Compute average load_duration_s from successful runs. Return 0 if no data.

    3. `_get_sort_value(model_data: dict, sort_by: str) -> float`: If sort_by in ["response_ts", "total_ts", "prompt_eval_ts"], return from model_data["averages"]. If sort_by == "load_time", return _compute_load_time_avg(model_data). Return 0 if key missing.

    4. `analyze_results(filepath: str, sort_by: str = "response_ts", top_n: int | None = None, ascending: bool = False, detail: bool = False) -> None`:
       - Load JSON from filepath (use Path, handle FileNotFoundError and JSONDecodeError with console error messages and return)
       - Get models list, handle missing "mode" field gracefully (default to "standard")
       - Sort models by _get_sort_value(m, sort_by), reverse=not ascending
       - If top_n, slice to first top_n models
       - Build Rich Table with title "Results: {filename}" (just the stem)
       - Columns: Rank (#), Model, Response (t/s), Total (t/s), Prompt Eval (t/s), plus Load Time (s) if sort_by == "load_time"
       - Bold the sorted column header
       - Add row per model with averages values
       - If detail=True: add indented rows per run under each model showing per-run metrics (dim style, no rank number)
       - Print table via get_console()

    Create tests/test_analyze.py with sample JSON fixtures:

    - Fixture: sample_results_json — a tmp_path JSON file with 3 models, each with 2 runs, varying response_ts values
    - test_sort_by_response_ts: Default sort, verify first model has highest response_ts
    - test_sort_ascending: ascending=True, verify first model has lowest response_ts
    - test_top_n_filter: top_n=2, verify only 2 models in output
    - test_sort_by_load_time: sort_by="load_time", verify sorting uses load_duration_s averages
    - test_invalid_sort_key: sort_by="invalid" raises or prints error gracefully
    - test_file_not_found: Non-existent file prints error, doesn't crash
    - test_detail_mode: detail=True doesn't crash (verify by calling without exception)

    Tests should capture console output using rich.console.Console(file=StringIO) or by checking that the function completes without error. For sort verification, test _get_sort_value directly and test the sorted order by inspecting the model data before Rich rendering.
  </action>
  <verify>
    <automated>uv run pytest tests/test_analyze.py -x -v</automated>
  </verify>
  <done>analyze_results sorts by any valid metric, filters top-N models, supports detail mode. Graceful error handling. All tests pass.</done>
</task>
