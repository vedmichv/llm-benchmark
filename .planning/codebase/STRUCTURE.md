## Directory Purposes

**Root Directory:**
- Purpose: Python package root; contains all executable scripts and configuration
- Contains: 5 Python entry points, 3 launcher scripts, docs, git metadata
- Key files: `run.py` (primary), `benchmark.py` (simple), `extended_benchmark.py` (full)

**.planning/codebase/:**
- Purpose: GSD-generated architecture and quality documentation
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: `/gsd:map-codebase` command

**__pycache__/:**
- Purpose: Python interpreter cache for .pyc files
- Generated: Automatic on import
- Committed: No (.gitignore excludes)

**.venv/:**
- Purpose: Optional isolated Python environment
- Generated: By `run.py` via `python -m venv`
- Committed: No (.gitignore excludes)

## Key File Locations

**Entry Points:**
- `run.py`: Cross-platform launcher (Windows, Linux, macOS) - checks Python/Ollama/dependencies, creates venv, delegates to extended_benchmark.py
- `benchmark.py`: Simple single-file benchmark - quick runs with default prompts, stdout-only output
- `extended_benchmark.py`: Production benchmark - system profiling, model offloading, multi-format output (MD/JSON/CSV), timeout handling
- `compare_results.py`: Post-analysis tool - loads JSON results, displays comparison tables

**Configuration:**
- `requirements.txt`: Lists Python dependencies (ollama, pydantic) - no version pinning
- `.gitignore`: Excludes cache, venv, .env files, generated results
- `CLAUDE.md`: Developer instructions, architecture overview, setup commands

**Core Logic:**
- `benchmark.py` (Lines 42-216): `run_benchmark()`, `inference_stats()`, `average_stats()`, `get_benchmark_models()`, `main()`
- `extended_benchmark.py` (Lines 220-1050+): `collect_system_info()`, `benchmark_model()`, `run_single_benchmark()`, `save_results_to_*()`, `main()`
- `run.py` (Lines 60-331): `check_python()`, `check_ollama()`, `setup_venv()`, `check_dependencies()`, `run_benchmark()`

**Testing:**
- `test_ollama.py`: Manual connectivity test (not automated test suite) - verifies ollama CLI, lists models, runs sample prompt

**Documentation:**
- `README.md`: Feature overview, setup instructions, usage examples, troubleshooting
- `CLAUDE.md`: Architecture summary, prerequisites, running benchmarks, development notes
- `PROGRESS.md`: Development roadmap and completed features
- `WINDOWS_SETUP.md`: Step-by-step Windows installation with WSL notes
- `WSL_SETUP.md`: WSL-specific configuration

**Launcher Scripts:**
- `run.sh`: Unix shell wrapper around `python run.py`
- `run.bat`: Windows batch wrapper around `python run.py`
- `run.ps1`: Windows PowerShell wrapper around `python run.py`

**Utility Scripts:**
- `setup_passwordless_sudo.sh`: Linux helper for model offloading without password prompt

## Naming Conventions

**Files:**
- `benchmark.py`, `extended_benchmark.py`: Lowercase with underscores for multi-word names
- `compare_results.py`: Descriptive verb-noun pattern
- `test_ollama.py`: test_* prefix for manual testing utilities
- `*_setup*.sh`, `*.bat`, `*.ps1`: Platform-specific scripts clearly named
- Output files: `benchmark_results_YYYYMMDD_HHMMSS.{md,json,csv}` - timestamp in filename

**Functions:**
- `run_benchmark()`, `run_single_benchmark()`: Verb-noun pattern indicating action
- `collect_system_info()`, `check_ollama()`: Verb-noun with clear intent
- `get_all_models()`, `get_benchmark_models()`: Getter pattern for retrieval
- `nanosec_to_sec()`: Helper-like naming for utility conversions
- `save_results_to_markdown()`, `save_results_to_json()`: Explicit destination naming
- Private/internal functions: Prefixed with `_` (e.g., `_run_streaming()`)

**Variables:**
- `model_name`, `prompt`, `prompts`: Descriptive snake_case
- `prompt_eval_ts`, `response_ts`, `total_ts`: Abbreviated metrics with clear suffix
- `PROMPT_SETS`: Module-level constant in UPPER_CASE
- `LOCK_FILE`, `BenchmarkTimeoutError`: Class-like constants in UPPER_CASE

**Classes & Models:**
- `Message`, `OllamaResponse`, `BenchmarkResult`: PascalCase for Pydantic models
- `ModelBenchmarkSummary`, `SystemInfo`: Descriptive PascalCase combining concepts
- `BenchmarkTimeoutError`, `Colors`: Exception and utility classes in PascalCase

**Arguments:**
- `--verbose`, `--skip-models`, `--prompt-set`: Lowercase with hyphens (argparse convention)
- `--runs-per-prompt`, `--no-offload`, `--export-json`: Descriptive long-form flags

## Special Directories

**__pycache__/:**
- Purpose: Bytecode caching for Python performance
- Generated: Automatic on first import
- Committed: No - excluded by .gitignore

**.venv/:**
- Purpose: Isolated Python environment created by `run.py` for dependency isolation
- Generated: By `python -m venv .venv` in `setup_venv()` (line 125)
- Committed: No - excluded by .gitignore
- Fallback: Script continues if venv unavailable (Debian/Ubuntu missing python3-venv)

**.git/:**
- Purpose: Git repository metadata and history
- Generated: By `git init` or clone
- Committed: N/A - special directory

**.planning/codebase/:**
- Purpose: GSD-generated documentation for architecture and quality
- Generated: By `/gsd:map-codebase` command
- Committed: Yes - documentation for reference

**Generated Output Files:**
- Location: Root directory (same as run.py)
- Pattern: `benchmark_results_YYYYMMDD_HHMMSS.{md,json,csv}`
- Committed: No - typically excluded or in .gitignore
- Cleanup: User responsibility or scripted separately
