  <name>Task 1: Add warmup and retry logic to runner with tests</name>
  <files>llm_benchmark/config.py, llm_benchmark/runner.py, tests/test_runner.py</files>
  <behavior>
    - TestWarmupModel: warmup_model() calls ollama.chat with short prompt and model name
    - TestWarmupModel: warmup_model() returns True on success, False on exception
    - TestWarmupModel: warmup_model() prints "Warming up {model}..." then "Ready" on success
    - TestWarmupModel: warmup_model() prints failure message on exception (does not raise)
    - TestRetryLogic: run_single_benchmark retries on ConnectionError (up to max_retries)
    - TestRetryLogic: run_single_benchmark retries on TimeoutError
    - TestRetryLogic: run_single_benchmark retries on ollama.RequestError
    - TestRetryLogic: run_single_benchmark retries on ollama.ResponseError with status_code >= 500
    - TestRetryLogic: run_single_benchmark does NOT retry on ollama.ResponseError with status_code 404
    - TestRetryLogic: run_single_benchmark with max_retries=0 does not retry
    - TestRetryLogic: after retries exhausted, returns BenchmarkResult(success=False) with error message
    - TestBenchmarkModelWarmup: benchmark_model calls warmup_model before prompt loop when skip_warmup=False
    - TestBenchmarkModelWarmup: benchmark_model skips warmup and prints advisory when skip_warmup=True
  </behavior>
  <action>
1. In config.py, add two constants:
   - `DEFAULT_MAX_RETRIES: int = 3`
   - `DEFAULT_WARMUP_PROMPT: str = "Hello"`

2. In runner.py, add `warmup_model()` function:
   - Signature: `warmup_model(model_name: str, timeout: int = DEFAULT_TIMEOUT) -> bool`
   - Print "  Warming up {model_name}..." (no newline), call `ollama.chat(model=model_name, messages=[{"role": "user", "content": DEFAULT_WARMUP_PROMPT}])` wrapped in `run_with_timeout`
   - On success print " Ready" and return True
   - On any exception print " [yellow]Warmup failed: {exc}[/yellow]" and return False
   - Import DEFAULT_WARMUP_PROMPT from config

3. Modify `run_single_benchmark()` to accept `max_retries: int = DEFAULT_MAX_RETRIES` parameter:
   - Import tenacity: `retry, stop_after_attempt, wait_exponential, retry_if_exception, before_sleep`
   - Import `from ollama import RequestError, ResponseError`
   - Add `_is_retryable(exc)` helper: returns True for ConnectionError, TimeoutError, RequestError, and ResponseError with status_code >= 500
   - When `max_retries > 0`: build dynamic tenacity retryer with `stop_after_attempt(max_retries + 1)`, `wait_exponential(multiplier=1, min=1, max=10)`, `retry_if_exception(_is_retryable)`, and a `before_sleep` callback that prints "    [yellow]Retry {attempt}/{max_retries}...[/yellow]"
   - The retry wraps the `_run_benchmark()` inner function call. Each retry attempt gets its own `run_with_timeout` call (retry OUTSIDE timeout, so each attempt gets full timeout budget)
   - When `max_retries == 0`: call `_run_benchmark` directly via `run_with_timeout` as before
   - The outer try/except catches final failures and returns BenchmarkResult(success=False)

4. Modify `benchmark_model()` to accept `skip_warmup: bool = False` and `max_retries: int = DEFAULT_MAX_RETRIES`:
   - Before the prompt loop: if not skip_warmup, call warmup_model(model_name, timeout)
   - If skip_warmup, print "  [dim]Warmup skipped -- first run may include model load time[/dim]"
   - Pass max_retries through to run_single_benchmark

5. Write tests in tests/test_runner.py (add to existing file):
   - class TestWarmupModel: mock ollama.chat, test success/failure paths, verify console output
   - class TestRetryLogic: mock ollama.chat to raise various exceptions, verify retry behavior, verify non-retryable errors fail immediately
   - class TestBenchmarkModelWarmup: mock warmup_model and run_single_benchmark, verify warmup called/skipped

Key anti-patterns to AVOID:
- Do NOT put retry inside run_with_timeout (kills all retries on single timeout)
- Do NOT use decorator @retry (max_retries is runtime, not import-time)
- Do NOT retry 4xx ResponseError (model not found, etc.)
- warmup_model must NOT use run_single_benchmark (would pollute results)
