  <action>
Create llm_benchmark/comparison.py with these components:

**Data Models (Pydantic):**
- `BackendModelResult(BaseModel)`: backend (str), model (str), avg_response_ts (float), avg_prompt_eval_ts (float), avg_total_ts (float)
- `ComparisonResult(BaseModel)`: backends (list[str]), models (list[str]), results (list[BackendModelResult]), winner_per_model (dict[str, str]), overall_winner (str), overall_wins (dict[str, int])

**GGUF Matching (Claude's discretion):**
- `match_gguf_to_ollama_name(ollama_name: str, gguf_files: list[tuple[Path, str]]) -> Path | None`: Normalize both names (lowercase, strip dots/hyphens), substring match base name and size tag. Fallback to display_name match. Return None if no match.

**Orchestration:**
- `run_comparison(backends: list[BackendStatus], prompts, runs_per_prompt, timeout, skip_warmup, max_retries, verbose, skip_models, skip_checks) -> ComparisonResult`:
  - Loop backends sequentially. For each: create_backend(), run_preflight_checks() to get models, benchmark_model() for each model, unload_model() after each.
  - For llama-cpp: use scan_gguf_files() to find GGUFs, match to Ollama model names, set _model_path on backend. Skip unmatched models.
  - Collect all results into dict[backend_name, list[ModelSummary]].
  - Build ComparisonResult: compute winner_per_model (max avg_response_ts per model), overall_winner (most wins), overall_wins.
  - If only 1 backend in list: warn via console, run single-backend benchmark, still return ComparisonResult.

**Display:**
- `render_comparison_bar_chart(backend_rates: list[tuple[str, float]], model_name: str) -> None`: Same pattern as display.py render_bar_chart but title is "{model_name} - Backend Comparison", entries are backend names, star on fastest. Import BAR_FULL, BAR_EMPTY, BAR_WIDTH from display.py.
- `render_comparison_matrix(results: dict[str, list[ModelSummary]]) -> None`: Rich Table, models as rows, backends as columns, response t/s values. Winner column with backend name. "--" for missing combos. Star on per-row max value. "Fastest backend: X (N/M models)" summary at bottom.
- For single model across multiple backends: call render_comparison_bar_chart. For multiple models: call render_comparison_matrix. Decision made in run_comparison based on total unique models across all backends.

**Export:**
- `export_comparison_json(comparison: ComparisonResult, system_info: SystemInfo, output_dir="results") -> Path`: Use _ensure_dir, _timestamp. Filename: "comparison_{timestamp}.json". Include full ComparisonResult as dict plus system_info.
- `export_comparison_markdown(comparison: ComparisonResult, system_info: SystemInfo, output_dir="results") -> Path`: Filename: "comparison_{timestamp}.md". Markdown with header, system info line, matrix table in Markdown format, winner recommendation, and text bar charts per model.

Use get_console() for all output. Lazy-import detect_backends, create_backend, benchmark_model, etc. inside functions to avoid circular imports. Follow existing codebase patterns exactly.
