  <name>Task 1: Parameter sweep module</name>
  <files>llm_benchmark/sweep.py, tests/test_sweep.py</files>
  <behavior>
    - get_model_layers("model") calls ollama.show(), searches modelinfo keys for "*block_count", returns int or None
    - build_sweep_configs(block_count, has_gpu) returns list of (num_ctx, num_gpu) tuples: all SWEEP_NUM_CTX crossed with num_gpu values [0] + [block_count//2, block_count] if block_count and has_gpu
    - build_sweep_configs with no block_count or no GPU returns only num_ctx variations with num_gpu=0
    - run_sweep_for_model(model_name, timeout, skip_warmup) orchestrates: get layers, build configs, warmup once, run each config with SWEEP_PROMPT, collect SweepConfigResult list, pick best by response_ts, return SweepModelResult
    - Config counter printed: "Testing config 3/12: num_ctx=2048, num_gpu=16..."
    - Failed configs get SweepConfigResult(success=False, error=...), sweep continues
    - Model is unloaded between config changes (ollama may need reload for different options)
    - Rich table displayed at end showing all configs ranked, best highlighted
  </behavior>
  <action>
    Create llm_benchmark/sweep.py with:

    1. `get_model_layers(model_name: str) -> int | None`: Call ollama.show(model_name), iterate modelinfo items looking for key containing "block_count", return int(value) if found, None otherwise. Wrap in try/except (if show fails, return None).

    2. `build_sweep_configs(block_count: int | None, has_gpu: bool) -> list[tuple[int, int]]`: Cross SWEEP_NUM_CTX with num_gpu values. num_gpu values: always [0]. If block_count is not None AND has_gpu is True, add block_count // 2 and block_count. Return list of (num_ctx, num_gpu) tuples.

    3. `_run_single_config(model_name: str, num_ctx: int, num_gpu: int, timeout: int) -> SweepConfigResult`: Call ollama.chat() with messages=[{"role": "user", "content": SWEEP_PROMPT}], options=Options(num_ctx=num_ctx, num_gpu=num_gpu). Parse response with OllamaResponse. Compute response_ts and total_ts. Return SweepConfigResult. On exception, return SweepConfigResult(success=False, error=str(exc), response_ts=0, total_ts=0, eval_count=0, total_duration_s=0).

    4. `_display_sweep_table(sweep_result: SweepModelResult)`: Create Rich Table with columns: Rank, num_ctx, num_gpu, Response (t/s), Total (t/s), Tokens, Time(s). Sort configs by response_ts descending. Best row gets green bold style. Title: "Sweep Results: {model}".

    5. `run_sweep_for_model(model_name: str, timeout: int = DEFAULT_TIMEOUT, skip_warmup: bool = False) -> SweepModelResult`: Orchestrate the full sweep:
       - Detect GPU from system info (import _get_gpu_info from system.py or check SystemInfo)
       - Call get_model_layers() for block_count
       - Build configs via build_sweep_configs()
       - Warmup model once if not skip_warmup
       - For each config: print counter ("Testing config {i+1}/{total}: num_ctx={ctx}, num_gpu={gpu}..."), call _run_single_config(), then unload_model() to force clean reload
       - Pick best_config = max successful by response_ts (or None if all failed)
       - Print summary: "Best config: num_ctx={x}, num_gpu={y} @ {ts:.1f} t/s"
       - Call _display_sweep_table()
       - Return SweepModelResult

    Note: If SweepConfigResult/SweepModelResult models don't exist yet in models.py (Plan 01 may not have completed), create them directly in sweep.py temporarily. The executor should check models.py first and import if available.

    Create tests/test_sweep.py:
    - test_get_model_layers: Mock ollama.show() to return modelinfo with "llama.block_count": 32. Verify returns 32.
    - test_get_model_layers_missing: Mock ollama.show() with no block_count key. Verify returns None.
    - test_build_sweep_configs_with_gpu: block_count=32, has_gpu=True. Verify 4 num_ctx * 3 num_gpu = 12 configs.
    - test_build_sweep_configs_no_gpu: block_count=None, has_gpu=False. Verify 4 configs (num_ctx only, all num_gpu=0).
    - test_best_config_selection: Create SweepModelResult with multiple configs, verify best_config has highest response_ts.
    - test_failed_config_continues: Mock chat to fail for one config. Verify sweep continues and that config has success=False.
